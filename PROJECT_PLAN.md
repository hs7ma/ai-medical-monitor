# نظام المراقبة الطبية المدعوم بالذكاء الاصطناعي

> **AI-Powered Medical Monitor System**
> نظام مراقبة وتشخيص طبية متعدد الأسرّة للمستشفيات، مدعوم بالذكاء الاصطناعي، باستخدام ESP32 وحساسات MAX30102 و MLX90614 لجمع العلامات الحيوية اللحظية، مع تطبيق ويب ثنائي اللغة (عربي/إنجليزي) يعتمد على OpenAI API (نصي + بصري) لتشخيص الأمراض المساعد، يشمل رفع فحوصات وتحاليل (صور/PDF) لتحليلها بالذكاء الاصطناعي إلى جانب القراءات الحيوية.

## 1. نظرة عامة

### 1.1 وصف المشروع

نظام مراقبة وتشخيص طبي ذكي مخصص للاستخدام في المستشفيات (متعدد الأسرّة)، يقوم بجمع القراءات الحيوية للمريض بشكل لحظي ومستمر من حساسات متصلة بـ ESP32، كما يتيح للأطباء رفع الفحوصات المخبرية والتحاليل والصور الطبية (PDF/صور) ليتم تحليلها بواسطة OpenAI (نص + رؤية) إلى جانب العلامات الحيوية، ثم يبثها إلى خادم مركزي يخزّنها ويعرضها على لوحة تحكم ويب ثنائية اللغة، مع تقديم تحليل وتشخيص مساعد شامل للطاقم الطبي.

### 1.2 مصادر التشخيص

| المصدر | الوصف | معدل التحديث |
|---|---|---|
| العلامات الحيوية (ESP32) | SpO₂، نبض القلب، حرارة الجسم | مستمر (لحظي) |
| الفحوصات المخبرية (رفع يدوي) | صور تحاليل الدم، البول، إلخ (PDF/صور) | عند الطلب |
| الصور الطبية (رفع يدوي) | أشعة X، CT، MRI، ultrasound (PDF/صور) | عند الطلب |
| تقارير سابقة (رفع يدوي) | تقارير طبية سابقة (PDF) | عند الطلب |

> الذكاء الاصطناعي يجمع بين **العلامات الحيوية اللحظية** و**الفحوصات المرفوعة** لتقديم تشخيص مساعد متكامل.

### 1.4 القياسات الحيوية المرصودة

| القياس | الحساس | الوحدة | معدل التحديث |
|---|---|---|---|
| نسبة الأكسجين في الدم (SpO₂) | MAX30102 | % | مستمر |
| معدل ضربات القلب (HR) | MAX30102 | bpm | مستمر |
| حرارة الجسم | MLX90614 | °C | كل 5 ثوانٍ |

### 1.5 الفئة المستهدفة

- **المستخدمون:** أطباء، ممرضون، مديرو نظام في مستشفى متعدد الأسرّة.
- **المرضى:** مراقبة مركزية لعدة أسرّة في وقت واحد.

| الاتصال | البروتوكول | السبب |

| الخادم ↔ المتصفح | WebSocket | بث لحظي ثنائي الاتجاه |
| الخادم ↔ OpenAI | HTTPS REST | استدعاء آمن للتحليل عند الطلب |
| الخادم ↔ قواعد البيانات | TCP داخلي | اتصال داخل شبكة Docker |

## 4. مكونات العتاد (Hardware)

### 4.1 لكل سرير

| المكوّن | الوظيفة | ناقل الاتصال | العنوان I²C |
|---|---|---|---|
| ESP32 (ESP32-WROOM-32) | وحدة المعالجة والاتصال بالشبكة | — | — |
| MAX30102 | قياس SpO₂ وضربات القلب | I²C | 0x57 |
| MLX90614 | قياس حرارة الجسم (لا تلامس) | I²C | 0x5A |
| SSD1306 OLED (0.96") | عرض محلي للقراءات على السرير | I²C | 0x3C |

> **ملاحظة:** جميع الحساسات على نفس ناقل I²C بعناوين مختلفة — لا تعارض.

### 4.2 مخطط التوصيل

```
ESP32
 ├── GPIO 21 (SDA) ─┬── MAX30102 SDA
 │                  ├── MLX90614 SDA
 │                  └── SSD1306 SDA
 ├── GPIO 22 (SCL) ─┬── MAX30102 SCL
 │                  ├── MLX90614 SCL
 │                  └── SSD1306 SCL
 ├── 3.3V ──────────┬── VCC (جميع الحساسات)
 └── GND ───────────┴── GND (جميع الحساسات)
```

### 4.3 مصدر الطاقة

- تشغيل من مصدر طاقة المستشفى (USB 5V → 3.3V داخلي في ESP32).
- لا حاجة لبطارية في بيئة المستشفى.

## 9. تكامل الذكاء الاصطناعي (OpenAI)

### 9.1 نموذج الاستخدام

- **النموذج:** GPT-4o (يدعم النص والرؤية — Vision) أو الأحدث المتاح.
- **وضع الاستدعاء:** عند الطلب فقط (عند ضغط الطبيب على "تشخيص") — لتقليل التكلفة.
- **الخصوصية:** إرسال أرقام ومعرّفات وهمية فقط، لا أسماء أو هويات حقيقية. الصور يتم إخفاء أي بيانات تعريفية (DICOM metadata) قبل الإرسال.

### 9.2 وضعا التشخيص

#### 9.2.1 تحليل العلامات الحيوية (Text-only)

تحليل القراءات اللحظية والتاريخية من الحساسات (SpO₂، نبض، حرارة) لاستنتاج الاتجاهات والحالات المحتملة.

#### 9.2.2 التشخيص الشامل (Vision + Text) — الميزة الأساسية

تحليل متكامل يجمع بين:
- **العلامات الحيوية** اللحظية والتاريخية من ESP32.
- **نص مستخرج** من ملفات PDF المرفوعة (تحاليل مخبرية، تقارير).
- **الصور الطبية** المرفوعة (أشعة X، CT، MRI، ultrasound) — ترسل مباشرة إلى OpenAI Vision للتحليل البصري.

**مخطط تدفق التشخيص الشامل:**

```
الطبيب يختار مصادر التشخيص
        │
   ┌────┴─────┬───────────────┬───────────────┐
   ▼          ▼               ▼               ▼
علامات      ملفات PDF      صور طبية      نصوص مستخرجة
حيوية      (تحاليل)      (X/CT/MRI)      (OCR)
   │          │               │               │
   │    pdfplumber       OpenAI Vision   pytesseract
   │    (استخراج نص)     (تحليل بصري)    (OCR للصور)
   │          │               │               │
   └──────────┴───────┬───────┴───────────────┘
                      ▼
              ┌───────────────┐
              │   OpenAI      │
              │   GPT-4o      │
              │   (Function   │
              │   Calling)    │
              └───────┬───────┘
                      ▼
              تقرير تشخيص مساعد
              (severity + causes +
               recommendations +
               red_flags)
```

### 9.3 تقنيات التكامل

#### 9.3.1 Function Calling

هيكلة مخرجات التشخيص بصيغة JSON موثوقة:

```json
{
  "severity": "moderate",
  "vitals_summary": "SpO₂ منخفضة (92%)، نبض مرتفع (115 bpm)",
  "lab_findings": "WBC مرتفع (14.2)، CRP مرتفع (45)",
  "imaging_findings": "ظل رئوي يميني قد يشير لالتهاب رئوي",
  "possible_causes": ["عدوى بكتيرية محتملة", "التهاب رئوي يميني"],
  "recommendations": [
    "بدء مضاد حيوي تجريبي بعد زراعة الدم",
    "إجراء صورة صدر بدقة أعلى",
    "مراقبة مستمرة للنبض و SpO₂"
  ],
  "red_flags": [
    "إذا انخفض SpO₂ تحت 88% يجب الأكسجين فوراً",
    "إذا تجاوزت الحرارة 39°C يجب خافض حرارة"
  ],
  "confidence": "medium"
}
```

#### 9.3.2 معالجة الملفات المرفوعة

| نوع الملف | التقنية | المعالجة |
|---|---|---|
| PDF (تحاليل/تقارير) | pdfplumber / PyMuPDF | استخراج النص كاملاً وإرساله كسياق نصي |
| صورة (تحاليل/تقرير مكتوب) | pytesseract (OCR) | استخراج النص من الصورة |
| صورة طبية (X-ray/CT/MRI) | OpenAI Vision (GPT-4o) | إرسال الصورة مباشرة للتحليل البصري |
| PDF يحتوي صور طبية | PyMuPDF (تحويل لصور) + Vision | استخراج كل صفحة كصورة وإرسالها لـ Vision |

> **ملاحظة:** يتم تكبير وتحسين جودة الصور الطبية (contrast/sharpen) قبل إرسالها لـ Vision لرفع دقة التحليل.

#### 9.3.3 RAG (Retrieval-Augmented Generation)

- قاعدة معرفة طبية محلية تحتوي على إرشادات موثوقة (WHO، ACC، AHA، CDC).
- توليد **embeddings** للإرشادات وتخزينها في قاعدة بيانات متجهات (مثل ChromaDB).
- عند التشخيص، استرجاع الإرشادات ذات الصلة وإرفاقها كسياق للنموذج.

#### 9.3.4 كشف الشذوذ (Anomaly Detection)

- مقارنة القراءات الحالية بالاتجاه التاريخي للمريض.
- تنبيه ذكي عند انحراف غير معتاد (وليس فقط تجاوز عتبة ثابتة).
- يمكن تطبيق إحصائيات بسيطة (Z-score) أو نموذج تعلم آلي خفيف.

### 9.4 استراتيجية تقليل التكلفة

| الإجراء | الأثر |
|---|---|
| استدعاء عند الطلب فقط | تجنب الاستدعاء المستمر |
| كاش (Cache) للنتائج المتشابهة | تقليل الاستدعاءات المتكررة |
| Rate Limiting لكل مستخدم | منع الاستخدام المفرط |
| استخدام نافذة سياق محدودة | تقليل التوكنات المستهلكة |
| إرسال ملخصات بدل البيانات الخام | تقليل حجم الإدخال |
| تقليل دقة الصور قبل الإرسال | تقليل التوكنات البصرية |
| استخراج النص من PDF محلياً بدل Vision | توفير استدعاءات Vision للصور الطبية فقط |

### 11.1 الأمان التقني

| المجال | الإجراء |
|---|---|
